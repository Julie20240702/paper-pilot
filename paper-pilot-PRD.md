## 一、产品概述

### 产品名称
Paper Pilot

### 一句话定位
专为留学生和文科/社科研究者设计的 AI 论文阅读助手——打开 PDF，不离开这个界面，完成从"读懂"到"能用"的全过程。

### 解决的核心痛点
用户现在的工作流是：Google Scholar 找文章 → 复制摘要到 ChatGPT → 再回来打开 PDF → 复制某段原文到 ChatGPT 问问题 → 再回 PDF 找证据。这个过程在两个窗口之间来回切换，繁琐且低效。

Paper Pilot 把 AI 问答和 PDF 阅读合并在同一个界面，AI 回答时直接标注页码，点击跳转，用户不需要离开页面。

### 目标用户
- 在美国/海外读书的中国留学生（本科 + 研究生）
- 需要做文献综述的学生
- 需要读英文学术论文但非母语障碍较大的用户
- 主要涉及领域：人文社科、AI 方向

## 二、MVP 功能范围（第一版，1周内完成）

> 原则：只做核心流程，不做多余功能。能跑通"上传→分析→问答→跳转"这条主线就算成功。

### 2.1 界面布局
- 左右分栏布局
- 左边：PDF 阅读器（占约 55% 宽度）
- 右边：AI 功能面板（占约 45% 宽度）
- 中间有可拖动的分割线，用户可以自由调整左右比例
- 整体风格：简洁、学术感、深色/浅色均可

### 2.2 PDF 阅读器（左侧）
- 支持上传本地 PDF 文件（点击上传或拖拽）
- 显示 PDF 内容，支持多页
- 翻页功能：上一页 / 下一页按钮，显示"第 X 页 / 共 Y 页"
- 支持放大 / 缩小（按钮或滚轮）
- 支持从 AI 面板点击页码后自动跳转到对应页
- 使用 react-pdf 库实现

### 2.3 AI 功能面板（右侧）
右侧顶部有 4 个标签页：

#### 标签1：论文概览
用户上传 PDF 后自动生成，包含：
- 研究领域（一句话）
- 核心研究问题
- 研究方法（实验/调查/理论分析等）
- 主要结论（2-4 条）
- 作者承认的研究局限性

#### 标签2：关键论点
用户上传 PDF 后自动生成，包含：
- 列出 3-6 个主要论点
- 每个论点后标注来源页码，格式为可点击的 `[第X页]`，点击跳转左侧 PDF
- 每个论点下方附：AI 对该论点的一句话解释（用中文，帮助理解）

#### 标签3：图表清单
用户上传 PDF 后自动生成，包含：
- 列出论文中所有值得关注的图表
- 每个图表说明：在第几页、图表内容描述、适合用在什么场景（PPT/文献综述）
- 标注页码可点击跳转

#### 标签4：AI 问答
- 对话框界面，用户输入问题，AI 回答
- AI 回答中所有引用原文的地方，以 `[第X页]` 格式标注，可点击跳转
- 支持以下典型问题类型：
  - "这篇论文和我研究的主题 [XXX] 有关系吗？"
  - "这个实验具体怎么做的？"
  - "[某个专业术语] 是什么意思？"
  - "这个结论的原文证据在哪里？"
  - "这篇文章我可以怎么写进我的文献综述？"
- 用户可以针对 AI 已生成的论点/概览内容追问："这里对应原文哪里？"

### 2.4 翻译功能
- 用户在 PDF 左侧选中一段文字后，右侧出现"翻译"按钮
- 点击后 AI 给出中文翻译 + 专业术语解释
- 翻译结果显示在右侧面板

### 2.5 笔记功能
- 右侧面板底部有可折叠的笔记区域
- 用户可以随时输入笔记
- 笔记与当前 PDF 文件名绑定，保存在浏览器 localStorage
- 用户下次打开同名文件时，笔记仍然存在

## 四、技术栈

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| 前端框架 | React + Vite | 快速搭建，生态好 |
| 样式 | Tailwind CSS | 快速写样式 |
| PDF 渲染 | react-pdf | 在浏览器里显示 PDF |
| PDF 文字提取 | pdf-parse（后端）| 提取文字内容给 AI 读 |
| AI 调用 | Claude API (claude-sonnet-4-6) | 核心 AI 能力 |
| 后端 | Node.js + Express | 处理 PDF 解析和 API 调用 |
| 数据存储 | localStorage（浏览器本地）| 第一版不用数据库 |
| 部署 | Vercel | 免费，一键部署 |

## 六、核心用户流程

1. 用户打开网页，看到上传区域
2. 用户上传 PDF 文件
3. 系统自动开始解析，右侧显示加载动画
4. 解析完成，"论文概览"和"关键论点"自动生成
5. 用户浏览关键论点，点击 `[第X页]` 跳转到原文
6. 用户切换到"AI 问答"，输入自己的问题
7. AI 回答，带页码标注，用户点击跳转验证
8. 用户在笔记区记录重点
9. 用户选中 PDF 中某段文字，点击翻译，获得中文解释

## 七、AI Prompt 设计原则

所有 AI 输出必须遵循以下规则：
- 凡是引用原文内容，必须在末尾标注 `[第X页]`
- 页码标注必须准确，不能编造
- 论点解释优先用中文，帮助用户理解
- 回答要简洁，不要冗长
- 不确定的内容明确说"原文未提及"，不要编造
